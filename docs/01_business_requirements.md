# ビジネス要件

ReviewFlow は、差し戻し前提の申請業務を、フォーム定義・承認フロー定義・進捗管理・修正依頼・再提出まで一元管理できるSaaSである。

## ターゲット
- 小規模〜中規模組織
- Excel / メール / 紙運用が残っている組織
- 申請種別ごとに項目や承認ルールが異なる組織

## 提供価値
- テナントごとに独立運用できる
- 申請フォームを設定で増やせる
- フォームごとに承認フローを変えられる
- 差し戻し時に修正箇所を明示できる
- 修正が必要な項目だけ再入力させられる
- 進捗と履歴を一元管理できる

## ロール
- platform_admin
- tenant_admin
- approver
- applicant

## 重要な制約
- 1契約 = 1テナント = 1ワークスペース
- ユーザーの複数テナント所属はMVPでは未対応
- 承認フローは直列承認のみ
- 承認者は個人指定ではなくロール指定を基本とする
- 高度な条件分岐フォームはMVP対象外
