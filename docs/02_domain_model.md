# ドメインモデル

## Tenant
契約組織を表す。すべての業務データは Tenant に属する。

主な責務:
- 契約組織の識別
- プラン管理
- テナント状態管理

## User
テナントに所属する利用者。

主な責務:
- 認証主体
- ロール保持
- 申請者 / 承認者 / 管理者としての振る舞い

## Invitation
メンバー招待を表す。**テナント管理者（tenant_admin）** が、自テナントにユーザーを追加するために発行する。

主な責務:
- 招待メール相当の管理
- 招待状態管理
- 招待時ロール保持
- 招待者（invited_by）としてテナント管理者を記録する想定

## FormDefinition
申請フォーム定義の親。

主な責務:
- フォームの識別
- 公開状態管理
- 対象テナント内でのフォーム種別管理

## FormField
フォームの1項目。

主な責務:
- 項目名
- 項目タイプ
- 必須設定
- 選択肢
- 表示順

## ApprovalFlow
フォームに紐づく承認フロー定義。

主な責務:
- 対象フォームに対する承認ルール管理
- ステップ集合の管理

## ApprovalStep
承認フロー上の1ステップ。

主な責務:
- 承認順序
- 承認者ロール
- 差し戻し可否

## Application
申請実体。

主な責務:
- フォームに基づく申請の作成
- ステータス管理
- 現在承認ステップ管理

## ApplicationFieldValue
申請ごとの入力値。

主な責務:
- フォーム項目ごとの値保持

## ApplicationApproval
承認処理履歴。

主な責務:
- 誰がいつ承認 / 差し戻し / 却下したか保持

## CorrectionRequest
差し戻しイベント。

主な責務:
- 差し戻し全体コメント
- 修正依頼状態管理

## CorrectionRequestItem
差し戻し対象フィールド。

主な責務:
- 修正対象フィールドの識別
- 項目別コメント保持

## ExportJob
CSV出力ジョブ。

主な責務:
- 非同期出力管理
- ファイル生成状態管理

## AuditLog
監査ログ。

主な責務:
- 重要操作の証跡保持
